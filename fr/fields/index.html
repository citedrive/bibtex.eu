BibTeX Fields

A single BibTeX entry can hold the following fields. Note which are optional, required or ignored.

ğŸ“„ï¸ address

The address-field contains the publisher's (or any other institution) address.

ğŸ“„ï¸ annote

BibTeX Field type

ğŸ“„ï¸ author

In the `author`-field, you can specify all contributors to the work you want to cite. BibTeX has several ways of specifying author names but expects in all cases a consistent and exact arrangement so that inaccurate information could lead to the unwanted output of the author's name.

ğŸ“„ï¸ booktitle

To cite an `incollection, i.e., a part of a book/collection with its own title, or to give a title to an article, part of an inproceedings, the booktitle-field is used to distinguish. At the same time, title contains the title of the book, or incproceedings`.

ğŸ“„ï¸ chapter

BibTeX field type: chapter

ğŸ“„ï¸ crossref

If you have several entries referring to the same proceedings, you can use crossref to specify fields common to several entries only once by defining crossref = . Where citationkey is the citation key of the cross-referenced entry.

ğŸ“„ï¸ doi

`doi is used to store any DOI (Digital Object Identifier of the International DOI Foundation). DOI is a digital identifier that is intended to be permanent and unique. It is used, among other things, and mostly for articles from scientific journals. It can be used in article, but more rarely also in chapter, book, or inproceedings / conference`. This field is optional in all cases.

ğŸ“„ï¸ edition

The edition of a book, e.g. "edition = " Entering an ordinal number is recommended by default, and the first letter is usually capitalised.

ğŸ“„ï¸ editor

In the `editor`-field, you can specify all editors to the work you want to cite. BibTeX has several ways of specifying editor names but expects in all cases a consistent and exact arrangement so that inaccurate information could lead to the unwanted output of the editor's name.

ğŸ“„ï¸ email

Email includes the email address of the stated authors if you want to indicate them. It's not a common field and might not be supported.

ğŸ“„ï¸ howpublished

The howpublished field is used with the @misc entry, typically as a workaround for a source type BibTeX does not support. The most common example is when citing web pages and specifying a URL.

ğŸ“„ï¸ institution

The institution field is used together with the @techreport entry to specify the name of the institution that issued the report.

ğŸ“„ï¸ journal

The name of the journal in which the journal was published.

ğŸ“„ï¸ month

The month of publication, or creation if unpublished.

ğŸ“„ï¸ note

The notes field can be used to store additional information about a particular BibTeX entry that may be relevant to the reader or that is needed for a particular entry type and citation style but does not fit in any other field.

ğŸ“„ï¸ number

Usually the issue number (or similar identifier) of a journal, magazine or the number of technical report.

ğŸ“„ï¸ organization

Usually, an institution, the name of the conference sponsor/organizer is used here.

ğŸ“„ï¸ pages

Page range of, for example, a book. You can separate these either with commas or double dashes.

ğŸ“„ï¸ publisher

The name of the publisher of the book being cited.

ğŸ“„ï¸ school

school is specified when a phdthesis or mastersthesis is cited and is the educational institution where the student wrote the dissertation. This field is required in both entry types.

ğŸ“„ï¸ series

The field can, for example, optionally be used to specify the book series in which the book (or similar) was published.

ğŸ“„ï¸ title

The title field is saved for the title of a work to be cited. LaTeX-specific writing, such as capitalisation, should be taken into account.

ğŸ“„ï¸ type

Within the BibTeX entry, the type field is used to declare an explicit type, i.e. for technical report "Research Note", or "Government Report".

ğŸ“„ï¸ volume

The 'volume' field is used to enter the volume of a journal or multi-volume book.

ğŸ“„ï¸ year

The year of publication, or creation if unpublished.
